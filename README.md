# Gandalf UI

The internal component library of **Unergy** — a design system built on top of [shadcn/ui](https://ui.shadcn.com/) and [Reka UI](https://reka-ui.com/), extending them with semantic colors, opinionated variants, and composable primitives for use across the platform.

Built with **Nuxt 4**, **Vue 3**, **Tailwind CSS v4**, and **TypeScript**.

---

## Architecture

Components are organized in clearly separated layers. **Do not mix concerns between layers.**

```
app/components/
  ui/             → Raw shadcn/ui primitives. Never edit these.
  gandalf/
    base/         → Atomic G-components. Generic, single-responsibility.
    kit/          → Composed components built from base/ primitives.
  blocks/         → App-level compositions. No business logic.
  features/       → Feature-specific UI (knows about domain entities).
```

| Layer | Description |
|---|---|
| `ui/` | Generated by shadcn/ui. Do not edit. Wrap in `gandalf/base/` if customization is needed. |
| `gandalf/base/` | Atomic, generic, single-responsibility components. Named `G<ComponentName>` (e.g. `GBadge`). Use CVA for variants. Always import from `@/components/ui/`, never directly from `reka-ui`. |
| `gandalf/kit/` | Higher-order components composed from `base/` primitives. Still generic, no business domain knowledge. |
| `blocks/` | Compositions of `gandalf/` components repeated 3+ times across the app. No business logic. |
| `features/` | Components that know about platform-specific types (`Factura`, `Cliente`, etc.). |

## Getting Started

```bash
# Install dependencies
bun install

# Start development server
bun dev

# Build for production
bun build

# Generate static site
bun generate
```

The documentation site runs at `http://localhost:3000/docs`.

## Design Decisions

### Composition over abstraction

Components expose **slots** rather than locking into a fixed layout. For example, `GDropdown` exposes `{ open, selectedOption }` via its trigger slot:

```vue
<GDropdown v-model="selected" :options="options">
  <template #default="{ open, selectedOption }">
    <GBadge class="cursor-pointer gap-1">
      {{ selectedOption?.label ?? 'Select' }}
      <ChevronDown :class="{ 'rotate-180': open }" />
    </GBadge>
  </template>
</GDropdown>
```

This is preferred over creating a `BadgeDropdown` component that only handles one specific case.

### Shared types

Common option types are defined in `app/components/gandalf/base/types.d.ts`:

```ts
export type Option = {
  label: string
  value: string | number
  disabled?: boolean
  color?: `#${string}`
}
```
